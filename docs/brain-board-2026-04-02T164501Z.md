# Vibestorm Brain Board

Timestamp: 2026-04-02T16:45:01Z

This is a structured suggestion board for future design choices, open options, and caution points.

## Architecture Suggestions

- Keep a strict layer split: transport -> protocol -> session/world -> presentation.
- Treat all inbound data as untrusted until parsed and normalized.
- Build an internal event bus early, even if it is just `asyncio.Queue` based.
- Keep a capture/replay path in mind from the start for packet and event debugging.
- Prefer read-only observability tools before write-heavy features.
- Make the first useful client a protocol inspector with state, not a UI-first app.

## Repo Suggestions

- Add a top-level `README.md` soon so repo entry is not buried in `docs/`.
- Add `spec/message-coverage.md` to track which message types are understood.
- Add `spec/capability-coverage.md` to track which capabilities are used and why.
- Add `docs/decision-log.md` once implementation starts.
- Add `test/fixtures/` early and check in representative payloads.

## Python-Specific Suggestions

- Use typed wrappers for UUIDs, region handles, circuit codes, and sequence numbers where practical.
- Do not pass raw dictionaries between modules if a dataclass can exist instead.
- Keep async boundaries explicit and narrow.
- Avoid mixing parsing logic with side effects such as logging, network send, or world mutation.
- Avoid global singleton clients or shared mutable module state.

## Protocol Suggestions

- Implement packet-header parsing and zerocode in isolation first.
- Build message-template loading before large message handlers.
- Add packet hex-dump utilities for debugging malformed traffic.
- Model reliable resend and ACK handling as explicit state machines.
- Treat LLSD as a constrained protocol need, not a generic serialization problem.

## Testing Suggestions

- Add unit tests for packet header parsing before any live integration attempts.
- Keep real captured payloads in fixtures with short annotations.
- Add replay tests that feed packets into the decoder without network activity.
- Add golden tests for login and LLSD payload parsing.
- Add one integration script per milestone rather than one giant demo command.

## Observability Suggestions

- Redact credentials, tokens, and session secrets by default.
- Log protocol transitions as structured events.
- Track packet sequence, ACK, resend, and disconnect reasons explicitly.
- Separate debug logs for raw transport from human-readable session summaries.

## Product Suggestions

- Make the first visible milestone a clean terminal session summary.
- Add a region object counter before attempting object rendering.
- Prefer coarse avatar and object presence over incomplete visual fidelity.
- Keep inventory read-only when first introduced.

## Agent-Handoff Suggestions

- End each meaningful task with a handoff note using the repo template.
- Record why a library was rejected when the choice is non-obvious.
- Write docs for any protocol behavior that took more than trivial effort to infer.
- Preserve TODOs only when they name a concrete next action.

- whether to separate protocol-core from UI into distinct Python packages
- whether a later renderer should live in-process or out-of-process
- whether live sessions should be recordable for offline world-state replay
