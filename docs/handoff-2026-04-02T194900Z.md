# Handoff

## Summary

- moved Vibestorm from protocol pieces to a real local OpenSim connection path
- local OpenSim user creation is now repeatable
- Vibestorm now performs live bootstrap, capability resolution, event-queue polling, and first UDP handshake probing

## Current State

- XML-RPC login bootstrap works against local OpenSim
- seed capability resolution works
- `EventQueueGet` poll works and returns the expected empty state
- UDP `UseCircuitCode` and `CompleteAgentMovement` probes reach the simulator
- live replies decoded include `PacketAck`, `RegionHandshake`, `AgentMovementComplete`, `ParcelOverlay`, and `ObjectUpdate`
- the client is not yet maintaining a stable long-lived viewer session

## Decisions Made

- used stdlib XML-RPC and urllib for live bootstrap and capability work so the code runs without extra environment setup
- used a fixed local OpenSim avatar UUID for repeatable scripted user creation
- treated OpenSim as the live integration target before touching Linden-hosted infrastructure

## Open Questions

- whether `RegionHandshakeReply` plus `AgentUpdate` is sufficient for stable session progression in OpenSim without more ACK state
- how much reliable resend bookkeeping is needed before the session becomes durable
- whether `ObjectUpdate` decoding or `AgentUpdate` cadence is the next critical-path item

## Next Recommended Step

- implement a session object that keeps one UDP socket open, sends `RegionHandshakeReply`, then starts periodic `AgentUpdate`
