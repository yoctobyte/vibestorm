# Handoff Template

Use this file structure for any meaningful work handoff between agents or contributors.

Preferred destination for day-to-day handoff updates:

- `docs/current-handoff.md`

Use this template when a fresh structured handoff document is warranted.

## Summary

- what changed
- why it changed

## Current State

- what works
- what does not work
- what is unverified

## Files Touched

- list of paths

## Decisions Made

- decision
- reason
- source reference if relevant

## Open Questions

- unanswered question
- impact if unresolved

## Next Recommended Step

- one concrete next task

## Verification

- tests run
- scripts run
- what was not verified
